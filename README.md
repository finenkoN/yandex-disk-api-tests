# Автоматизация тестирования API Яндекс.Диска

Проект содержит набор автоматизированных тестов для REST API сервиса Яндекс.Диск. 

## Стек технологий
- Python 3
- Pytest
- Requests

## Структура проекта
- `tests/conftest.py` — конфигурация тестов и фикстуры для подготовки и очистки данных.
- `tests/test_positive.py` — тесты успешных сценариев (GET, POST, PUT, DELETE).
- `tests/test_negative.py` — тесты обработки ошибок (400, 401, 404, 409).
- `requirements.txt` — зависимости проекта.
- `.env` — файл для OAuth-токена.

## Описание тестов

### Позитивные тесты (Happy Path)
- **GET /v1/disk/**: Получение общих данных о дисковом пространстве.
- **PUT /v1/disk/resources**: Создание папки. Валидация поддержки кириллицы и спецсимволов.
- **GET /v1/disk/resources**: Получение метаданных. Проверка полей `name` и `type` в JSON.
- **POST /v1/disk/resources/copy**: Копирование ресурса (обязательный метод POST по ТЗ).
- **DELETE /v1/disk/resources**: Удаление папки и проверка очистки пространства.

### Негативные тесты (Error Handling)
- **401 Unauthorized**: Запрос с отсутствующим или невалидным токеном.
- **400 Bad Request**: Передача некорректных параметров (пустой путь).
- **404 Not Found**: Запрос данных, удаление или копирование несуществующего ресурса.
- **409 Conflict**: Попытка создания дубликата папки.
- **409 Conflict**: Создание папки в несуществующей родительской директории.

## Установка и запуск

Клонировать репозиторий:
```bash
git clone https://github.com/finenkoN/yandex-disk-api-tests
```

Создать и активировать виртуальное окружение:
```bash
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для Linux/macOS:
source venv/bin/activate
```

Установить зависимости:
```bash
pip install -r requirements.txt
```
Создать файл .env в корне проекта и указать OAuth-токен:
```bash
YANDEX_TOKEN=ваш_токен
```
Запустить тесты:
```bash
pytest -v
```
