# Автоматизация тестирования API Яндекс.Диска

Проект содержит набор автоматизированных тестов для REST API сервиса Яндекс.Диск. 

## Стек технологий
- Python 3
- Pytest
- Requests

## Структура проекта
- `tests/conftest.py` — конфигурация тестов и фикстуры для подготовки и очистки данных.
- `tests/test_positive.py` — тесты успешных сценариев (GET, POST, PUT, DELETE).
- `tests/test_negative.py` — тесты обработки ошибок (400, 401, 404, 409).
- `requirements.txt` — зависимости проекта.
- `.env` — файл для OAuth-токена.

## Описание тестов

### Позитивные тесты
- **PUT**: Создание новой папки. Проверка поддержки кириллицы и пробелов в именах.
- **GET**: Получение метаинформации о папке. Валидация полей ответа (name, type).
- **POST**: Копирование существующей папки.
- **DELETE**: Удаление ресурса и проверка корректности статуса ответа.

### Негативные тесты
- **401 Unauthorized**: Запрос без авторизационного заголовка.
- **400 Bad Request**: Создание ресурса с пустым параметром пути.
- **404 Not Found**: Запрос данных о несуществующем ресурсе.
- **409 Conflict**: Попытка создания уже существующего ресурса.
- **409 Conflict**: Создание папки в несуществующей родительской директории.

## Установка и запуск

1. Клонировать репозиторий:
```bash
git clone https://github.com/finenkoN/yandex-disk-api-tests
```

Создать и активировать виртуальное окружение:
```bash
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для Linux/macOS:
source venv/bin/activate
```

Установить зависимости:
```bash
pip install -r requirements.txt
```
Создать файл .env в корне проекта и указать OAuth-токен:
```bash
YANDEX_TOKEN=ваш_токен
```
Запустить тесты:
```bash
pytest -v
```
